Задание 1

Почему код дает именно такие результаты?

var a = 1, b = 1, c, d;
c = ++a; alert(c); // 2
d = b++; alert(d); // 1
c = (2+ ++a); alert(c); // 5
d = (2+ b++); alert(d); // 4
alert(a); // 3
alert(b); // 3

Объяснение:
var a = 1, b = 1, c, d;
c = ++a; alert(c); // 2 - сначала инкремент, потом вывод.
d = b++; alert(d); // 1 - сначала вывод, а потом уже инкремент.
c = (2+ ++a); alert(c); // 5 - сначала инкремент a (получится a = 3, так как до этого уже был инкремент a), потом к a прибавляется 2. Получается 5.
d = (2+ b++); alert(d); // 4 - сначала будет действие в скобках 2 + b (b = 2 из одной из предыдущих строк; так как инкремент постфиксный, то сначала вернётся значение b, а потом прибавится единица) = 2 + 2 = 4.
alert(a); // 3 - изначально a = 1, но на предыдущих строках были выполнены два его инкремента, по итогу получаем a = 3.
alert(b); // 3 - изначально b = 1, но на предыдущих строках были выполнены два его инкремента, по итогу получаем b = 3.

Задание 2

Чему будет равен x?

var a = 2;
var x = 1 + (a *= 2);

Объяснение: x будет равен 5. Сначала будет выполнено действие a = a * 2, потом к полученной 4 прибавится 1.

Задание 7

Сравнить null и 0. Объяснить результат.

Объяснение:
null > 0; // false, так как по итогу всех преобразований будет сравнение +0 и 0, что даст false.
null >= 0; // true, так как если null < 0 принимает значение false, то null >= 0 принимает значение true.
null < 0; // false, так как по итогу всех преобразований будет сравнение +0 и 0, что даст false.
null <= 0; // true, так как если null > 0 принимает значение false, то null <= 0 принимает значение true.
null == 0; // false, так как ни один алгоритм сравнения не подходит и возращается значение по умолчанию, то есть false.
null === 0; // false, строгое сравнение, сразу false.
